# COCOMO (Constructive Cost Model)

**Proposed by:** Barry W. Boehm (1981)  
**Source:** _Software Engineering Economics_

---

## Overview

The **Constructive Cost Model (COCOMO)** is a regression-based software estimation model used to predict:

- **Effort (E)** — measured in _person-months (PM)_
- **Development Time (D)** — measured in _months_
- **Average Staffing (SS)** and **Productivity (P)**

It estimates the time and effort required to develop software based on the project size (in Kilo Lines of Code, KLOC).

---

## Types of COCOMO Models

1. **Basic COCOMO**
2. **Intermediate COCOMO**
3. **Detailed COCOMO**

Each model increases in complexity and accuracy.

---

## 1. Basic COCOMO Model

### Formulae

\[
E = a_b \times (KLOC)^{b_b}
\]
\[
D = c_b \times (E)^{d_b}
\]

Where:

- **E** = Effort in Person-Months (PM)
- **D** = Development time in Months
- **KLOC** = Thousands of Delivered Source Lines of Code
- **a_b, b_b, c_b, d_b** = Mode-dependent constants

---

### Basic COCOMO Coefficients

| Mode              | a<sub>b</sub> | b<sub>b</sub> | c<sub>b</sub> | d<sub>b</sub> | Typical KLOC |
| :---------------- | :-----------: | :-----------: | :-----------: | :-----------: | :----------: |
| **Organic**       |      2.4      |     1.05      |      2.5      |     0.38      |     2–50     |
| **Semi-detached** |      3.0      |     1.12      |      2.5      |     0.35      |    50–300    |
| **Embedded**      |      3.6      |     1.20      |      2.5      |     0.32      |     >300     |

---

### Additional Derived Metrics

\[
SS = \frac{E}{D} \quad \text{(Average Staff Size)}
\]

\[
P = \frac{KLOC}{E} \quad \text{(Productivity in KLOC per PM)}
\]

---

## 2. Intermediate COCOMO Model

Includes **Cost Drivers (Effort Adjustment Factors)** to refine estimation.

### Formulae

\[
E = a_i \times (KLOC)^{b_i} \times EAF
\]

\[
D = c_i \times (E)^{d_i}
\]

Where:

- **EAF** = Effort Adjustment Factor (product of all cost driver multipliers)
- **aᵢ, bᵢ, cᵢ, dᵢ** = Coefficients for mode

### Intermediate COCOMO Coefficients

| Mode              | a<sub>i</sub> | b<sub>i</sub> | c<sub>i</sub> | d<sub>i</sub> |
| :---------------- | :-----------: | :-----------: | :-----------: | :-----------: |
| **Organic**       |      3.2      |     1.05      |      2.5      |     0.38      |
| **Semi-detached** |      3.0      |     1.12      |      2.5      |     0.35      |
| **Embedded**      |      2.8      |     1.20      |      2.5      |     0.32      |

---

### Effort Adjustment Factor (EAF)

\[
EAF = \prod\_{j=1}^{n} (Cost\ Driver)\_j
\]

Cost Drivers include:

- **Product Attributes:** Reliability, Complexity, Database Size
- **Hardware Attributes:** Performance, Memory, Turnaround Time
- **Personnel Attributes:** Capability, Experience
- **Project Attributes:** Tools, Modern Practices, Schedule Pressure

---

## 3. Detailed COCOMO Model

Adds **phase-sensitive effort multipliers** and considers software life cycle stages.

### Effort Allocation by Phase

| Lifecycle Phase       | Effort (%) | Time (%) |
| :-------------------- | :--------: | :------: |
| Plan & Requirements   |    6–8%    |  10–40%  |
| Design                |   16–18%   |  19–38%  |
| Programming           |   48–68%   |  24–64%  |
| Integration & Testing |   16–34%   |  18–34%  |

Effort for each phase:
\[
E*{phase} = E*{total} \times Effort\ Fraction
\]

Time for each phase:
\[
D*{phase} = D*{total} \times Time\ Fraction
\]

---

## 4. Example (Basic COCOMO)

For a **400 KLOC** project (Embedded mode):

\[
E = 3.6 \times (400)^{1.20} = 4772.81\ PM
\]

\[
D = 2.5 \times (4772.81)^{0.32} = 38\ months
\]

---

## 5. Example (Intermediate COCOMO)

For a **12 KLOC Organic** project:

\[
E = 3.2 \times (12)^{1.05} \times 1.2169 = 52.91\ PM
\]

\[
D = 2.5 \times (52.91)^{0.38} = 11.29\ months
\]

---

## Summary of Key Formulas

| Parameter                 | Formula                               | Description                           |
| :------------------------ | :------------------------------------ | :------------------------------------ |
| **Effort (E)**            | `E = a × (KLOC)^b × EAF`              | Total effort in person-months         |
| **Development Time (D)**  | `D = c × (E)^d`                       | Total development time in months      |
| **Average Staffing (SS)** | `SS = E / D`                          | Average team size                     |
| **Productivity (P)**      | `P = KLOC / E`                        | Productivity in KLOC/person-month     |
| **Effort per Phase**      | `E_phase = E_total × Effort_fraction` | Work distribution per lifecycle phase |
